2.1 Names and Objects

In C++ and Java, a variable is a named region of memory with a declared type. In Python, a variable is simply a name that references an object in memory. The type belongs to the object, not the name. This mechanism is called name binding.

x = 42          # x is bound to an int object
x = "hello"     # x is now bound to a str object - no error

There is no declaration step. The assignment = operator creates the binding. If the name already exists in the current scope, it is rebound to the new object.

Identity, Type, and Value

Every object in Python has three properties:

  • Identity — a unique integer returned by id(obj).
  • Type — returned by type(obj), determining supported operations.
  • Value — the data the object contains.
a = 256
b = 256
print(a is b)   # True - CPython caches small integers [-5, 256]

c = 1000
d = 1000
print(c is d)   # May be False - depends on implementation
Use == to compare values and is to compare identity (same object in memory). The is operator is analogous to comparing pointer addresses in C++.

2.2 Numeric Types

Python provides three built-in numeric types:

  • int — arbitrary-precision integers. Unlike C++ int (32 bits) or Java int (32 bits), Python integers grow as large as memory allows.
  • float — IEEE 754 double-precision floating-point, equivalent to double in C++/Java.
  • complex — complex numbers with real and imaginary parts (e.g., 3 + 4j).
big = 2 ** 200                  # Arbitrary precision - no overflow
pi = 3.141592653589793           # float
z = 2 + 3j                      # complex

print(type(big))   # <class 'int'>
print(type(pi))    # <class 'float'>

Arbitrary-Precision Integers — No Size Limit

One of the most striking differences between Python and C++/Java is that Python integers have no fixed size. They grow dynamically to accommodate any value, limited only by available memory. There is no overflow, no wraparound, and no need to choose between int, long, or long long.

import sys

# Small integer
a = 42
print(sys.getsizeof(a))      # 28 bytes

# Medium integer
b = 2 ** 100
print(b)                      # 1267650600228229401496703205376
print(sys.getsizeof(b))      # 44 bytes

# Large integer
c = 2 ** 1000
print(len(str(c)))            # 302 digits!
print(sys.getsizeof(c))      # 164 bytes

# Huge integer
d = 2 ** 10000
print(len(str(d)))            # 3011 digits
print(sys.getsizeof(d))      # 1356 bytes

# Python handles this without any special type
print(type(d))                # <class 'int'>  - still just "int"

As you can see, sys.getsizeof() reveals that Python integers grow in memory as the value increases. A small number like 42 uses 28 bytes, while 2 ** 10000 (a 3011-digit number) uses over 1 KB.

Comparison with C++ Integer Types

In C++, integer types have fixed sizes and maximum values. If a computation exceeds the maximum, the result silently overflows and wraps around — a common source of bugs.

C++ Fixed-Size Types

// C++ integer limits
#include <climits>
#include <iostream>

int main() {
    // int: 4 bytes, max 2,147,483,647
    std::cout << sizeof(int) << std::endl;        // 4
    std::cout << INT_MAX << std::endl;             // 2147483647

    // long long: 8 bytes, max 9.2 x 10^18
    std::cout << sizeof(long long) << std::endl;   // 8
    std::cout << LLONG_MAX << std::endl;           // 9223372036854775807

    // OVERFLOW! Silent wraparound
    int x = INT_MAX;
    std::cout << x + 1 << std::endl;  // -2147483648 (BUG!)
}

Python — No Limit

import sys

# Python int grows as needed
x = 2_147_483_647         # Same as C++ INT_MAX
print(x + 1)              # 2147483648 - no overflow!

x = 9_223_372_036_854_775_807  # Same as C++ LLONG_MAX
print(x + 1)              # 9223372036854775808 - still fine!

# Python can go far beyond any C++ type
x = 10 ** 100             # A googol - 101 digits
print(type(x))            # <class 'int'>
print(sys.getsizeof(x))   # 72 bytes

⚠ Memory Warning: Just because Python integers have no arithmetic limit does not mean they are free. Every digit costs memory. Computing extremely large values like 10 ** 10_000_000 can consume gigabytes of RAM and freeze your program. Always be mindful of the size of numbers in loops and recursive computations. The convenience of unlimited precision comes with the responsibility of managing memory.

for exp in [10, 100, 1000, 10_000, 100_000, 1_000_000]:

    size = sys.getsizeof(n)

    print(f"2^{exp:>10,}  →  {digits:>8,} digits  →  {size:>10,} bytes")

Arithmetic Operators

a, b = 17, 5

print(a + b)    # 22   Addition
print(a - b)    # 12   Subtraction
print(a * b)    # 85   Multiplication
print(a / b)    # 3.4  True division (always returns float)
print(a // b)   # 3    Floor division
print(a % b)    # 2    Modulo
print(a ** b)   # 1419857  Exponentiation
In C++ and Java, 17 / 5 performs integer division yielding 3. In Python, / always performs true division. Use // for floor division.

Readable Numeric Literals — Underscore Separators

Python allows underscores (_) inside numeric literals as visual separators. They are ignored by the interpreter and exist only to make large numbers easier to read:

# Without underscores (hard to read)
population = 8100000000
budget = 4500000.75

# With underscores (much clearer)
population = 8_100_000_000
budget = 4_500_000.75

print(population)   # 8100000000  (underscore is not stored)
print(budget)       # 4500000.75

# Works with any numeric type
binary  = 0b1010_0001_1100_0011
hex_val = 0xFF_EC_DA
octal   = 0o17_55
big_int = 10_000_000_000_000

# Also works with floats and scientific notation
pi_approx = 3.14_15_92
avogadro  = 6.022_140_76e23
Underscores in numbers were introduced in Python 3.6 (PEP 515). C++ added a similar feature (1'000'000) in C++14, but uses single quotes instead.

Number Bases: Binary, Octal, Decimal, and Hexadecimal

Python supports four integer bases. Decimal (base 10) is the default numbering system — any integer literal without a prefix is decimal.

BasePrefixDigitsExampleDecimal Value
Binary (2)0b or 0B0–10b101010
Octal (8)0o or 0O0–70o1715
Decimal (10)none (default)0–9255255
Hexadecimal (16)0x or 0X0–9, A–F0xFF255
# Writing integers in different bases
decimal = 255
binary  = 0b11111111     # 255 in binary
octal   = 0o377          # 255 in octal
hexa    = 0xFF           # 255 in hexadecimal

# They are all the same integer
print(decimal == binary == octal == hexa)   # True
print(type(binary))   # <class 'int'>  — always stored as int

Converting Between Bases

Use built-in functions to convert an integer to a string in a different base, or parse a string in a given base back to an integer:

n = 255

# int → string in different bases
print(bin(n))     # '0b11111111'
print(oct(n))     # '0o377'
print(hex(n))     # '0xff'

# Remove the prefix if needed
print(bin(n)[2:])   # '11111111'
print(hex(n)[2:])   # 'ff'

# String → int  (specify the base)
print(int('11111111', 2))    # 255   (binary string → int)
print(int('377', 8))         # 255   (octal string → int)
print(int('FF', 16))         # 255   (hex string → int)
print(int('255', 10))        # 255   (decimal string → int)

# int() auto-detects base when given a prefix and base=0
print(int('0b1010', 0))     # 10
print(int('0o17', 0))       # 15
print(int('0xFF', 0))       # 255

Formatted Output in Different Bases

n = 42

# Using format specifiers
print(f"{n:b}")     # '101010'    binary (no prefix)
print(f"{n:#b}")    # '0b101010'  binary with prefix
print(f"{n:o}")     # '52'        octal
print(f"{n:#o}")    # '0o52'      octal with prefix
print(f"{n:x}")     # '2a'        hex lowercase
print(f"{n:#X}")    # '0X2A'      hex uppercase with prefix
print(f"{n:d}")     # '42'        decimal (explicit)

# Combine with padding for aligned output
for i in [8, 16, 32, 64, 128, 255]:
    print(f"{i:>4d}  {i:#06x}  {i:#010b}")

Output:

#   16  0x0010  0b00010000
#   32  0x0020  0b00100000
#   64  0x0040  0b01000000
#  128  0x0080  0b10000000
#  255  0x00ff  0b11111111
Default numbering system: Python always uses decimal (base 10) by default. When you print() any integer, it is displayed in decimal, regardless of how it was written. The prefixes 0b, 0o, and 0x are only for literal notation and output formatting.

2.3 Strings

Strings are immutable sequences of Unicode characters. Python does not have a separate char type; a single character is simply a string of length one.

name = "Ada Lovelace"
initial = name[0]           # "A"
length = len(name)          # 12

upper = name.upper()        # "ADA LOVELACE"
parts = name.split()        # ["Ada", "Lovelace"]
joined = "-".join(parts)    # "Ada-Lovelace"

Indexing and Slicing

Python supports powerful indexing using square brackets []. You can use positive indices (counting from the start at 0) or negative indices (counting backwards from the end at -1).

text = "PYTHON"
#        P  Y  T  H  O  N
#        0  1  2  3  4  5    ← positive index
#       -6 -5 -4 -3 -2 -1   ← negative index

# Positive indexing (from the start)
print(text[0])     # "P"  - first character
print(text[3])     # "H"  - fourth character
print(text[5])     # "N"  - last character

# Negative indexing (from the end)
print(text[-1])    # "N"  - last character
print(text[-2])    # "O"  - second to last
print(text[-6])    # "P"  - first character

Slicing extracts a substring using the syntax string[start:stop:step]. The start is inclusive, stop is exclusive, and step is optional.

text = "PYTHON"

# Basic slicing [start:stop]
print(text[0:3])    # "PYT"  - characters 0, 1, 2
print(text[2:5])    # "THO"  - characters 2, 3, 4

# Omit start or stop
print(text[:3])     # "PYT"  - from beginning to index 3
print(text[3:])     # "HON"  - from index 3 to end
print(text[:])      # "PYTHON" - full copy

# Negative indices in slicing
print(text[-3:])    # "HON"  - last 3 characters
print(text[:-2])    # "PYTH" - everything except last 2

# Step (stride)
print(text[::2])    # "PTO"  - every 2nd character
print(text[1::2])   # "YHN"  - every 2nd character starting at 1
print(text[::-1])   # "NOHTYP" - reversed string!
print(text[::-2])   # "NHY"  - reversed, every 2nd character

Slicing is safe — it never raises an error for out-of-range indices. Python silently clips them to the valid range:

text = "PYTHON"

print(text[0:100])   # "PYTHON" - stop clipped to len(text)
print(text[-100:3])  # "PYT"    - start clipped to 0

# But direct indexing DOES raise an error
# text[100]  → IndexError: string index out of range
Key insight: Indexing and slicing work identically on all Python sequences — strings, lists, and tuples. Learn it once, use it everywhere. In C++, std::string::substr(pos, len) takes a position and a length, while Python slicing takes start and stop positions.

2.4 Booleans and Comparison Operators

The bool type has exactly two instances: True and False. Many values have an implicit truth value: zero, empty collections, and None are falsy; everything else is truthy.

# Comparison operators (same as C++/Java)
print(5 > 3)       # True
print(5 == 5.0)    # True - cross-type comparison

# Boolean operators (words, not symbols)
print(True and False)   # False
print(True or False)    # True
print(not True)         # False

# Chained comparisons (unique to Python)
x = 7
print(1 < x < 10)  # True - equivalent to (1 < x) and (x < 10)
Python uses and, or, not instead of &&, ||, !.

Short-Circuit Evaluation

Python's and and or operators use short-circuit evaluation — they stop evaluating as soon as the result is determined. This is the same concept as && and || in C++, but with an important twist: Python returns the actual value that decided the result, not just True or False.

and — Short-Circuits on First Falsy

If the left operand is falsy, return it immediately (the right operand is never evaluated). Otherwise return the right operand.

or — Short-Circuits on First Truthy

If the left operand is truthy, return it immediately (the right operand is never evaluated). Otherwise return the right operand.

# ── Short-circuit with `and` ──
# If the first value is falsy, Python returns it immediately
# and NEVER looks at the second value

print(0 and 42)           # 0     (0 is falsy → returned immediately)
print("" and "hello")     # ""    ("" is falsy → returned immediately)
print(None and 100)       # None  (None is falsy → returned immediately)

# If the first value is truthy, Python must check the second value
print(1 and 42)           # 42    (1 is truthy → return second value)
print("hi" and "hello")   # "hello"

# ── Short-circuit with `or` ──
# If the first value is truthy, Python returns it immediately
# and NEVER looks at the second value

print(42 or 0)            # 42    (42 is truthy → returned immediately)
print("hello" or "")      # "hello"

# If the first value is falsy, Python must check the second value
print(0 or 42)            # 42    (0 is falsy → return second value)
print("" or "default")    # "default"
print(None or "fallback") # "fallback"

Proving Short-Circuit: Skipped Evaluation

We can prove that the second operand is never evaluated by using a function with a side effect:

def expensive():
    print("  expensive() was called!")
    return 42

# With `and`: if first is falsy, expensive() is NEVER called
print("Test 1:")
result = 0 and expensive()
print(f"  result = {result}")
# Output:
#   Test 1:
#   result = 0               ← expensive() was NOT called

print("Test 2:")
result = 1 and expensive()
print(f"  result = {result}")
# Output:
#   Test 2:
#   expensive() was called!  ← expensive() WAS called
#   result = 42

# With `or`: if first is truthy, expensive() is NEVER called
print("Test 3:")
result = "exists" or expensive()
print(f"  result = {result}")
# Output:
#   Test 3:
#   result = exists           ← expensive() was NOT called

Practical Uses

Short-circuit evaluation is not just a curiosity — Python programmers use it every day:

# 1. Default values (very common pattern)
username = input_name or "Anonymous"
# If input_name is empty/""/None → use "Anonymous"

# 2. Safe access - avoid division by zero
x = 10
y = 0
safe = y != 0 and x / y    # y is 0 → False, x/y never computed
print(safe)                  # False (no ZeroDivisionError!)

# 3. Conditional function call
data = None
result = data and data.process()  # data is None → process() never called
# This avoids: AttributeError: 'NoneType' has no attribute 'process'

# 4. First available value from multiple sources
config = None
env_var = ""
default = "localhost"
host = config or env_var or default
print(host)  # "localhost" (first truthy value wins)
C++ comparison: C++ && and || also short-circuit, but they always return true or false (a boolean). Python returns the actual operand that determined the result, which makes patterns like value or default possible.

2.5 Type Conversion

Python performs limited implicit type promotion (e.g., int to float in mixed arithmetic). Explicit conversion uses constructor-like calls.

x = int("42")          # str to int
y = float(7)           # int to float
s = str(3.14)          # float to str
b = bool(0)            # int to bool - False

result = 10 + 2.5      # int + float = float (12.5)
