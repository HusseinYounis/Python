2.1 Names and Objects

In C++ and Java, a variable is a named region of memory with a declared type. In Python, a variable is simply a name that references an object in memory. The type belongs to the object, not the name. This mechanism is called name binding.

x = 42          # x is bound to an int object
x = "hello"     # x is now bound to a str object - no error

There is no declaration step. The assignment = operator creates the binding. If the name already exists in the current scope, it is rebound to the new object.

Identity, Type, and Value

Every object in Python has three properties:

  • Identity — a unique integer returned by id(obj).
  • Type — returned by type(obj), determining supported operations.
  • Value — the data the object contains.
a = 256
b = 256
print(a is b)   # True - CPython caches small integers [-5, 256]

c = 1000
d = 1000
print(c is d)   # May be False - depends on implementation
Use == to compare values and is to compare identity (same object in memory). The is operator is analogous to comparing pointer addresses in C++.

2.2 Numeric Types

Python provides three built-in numeric types:

  • int — arbitrary-precision integers. Unlike C++ int (32 bits) or Java int (32 bits), Python integers grow as large as memory allows.
  • float — IEEE 754 double-precision floating-point, equivalent to double in C++/Java.
  • complex — complex numbers with real and imaginary parts (e.g., 3 + 4j).
big = 2 ** 200                  # Arbitrary precision - no overflow
pi = 3.141592653589793           # float
z = 2 + 3j                      # complex

print(type(big))   # <class 'int'>
print(type(pi))    # <class 'float'>

Arbitrary-Precision Integers — No Size Limit

One of the most striking differences between Python and C++/Java is that Python integers have no fixed size. They grow dynamically to accommodate any value, limited only by available memory. There is no overflow, no wraparound, and no need to choose between int, long, or long long.

import sys

# Small integer
a = 42
print(sys.getsizeof(a))      # 28 bytes

# Medium integer
b = 2 ** 100
print(b)                      # 1267650600228229401496703205376
print(sys.getsizeof(b))      # 44 bytes

# Large integer
c = 2 ** 1000
print(len(str(c)))            # 302 digits!
print(sys.getsizeof(c))      # 164 bytes

# Huge integer
d = 2 ** 10000
print(len(str(d)))            # 3011 digits
print(sys.getsizeof(d))      # 1356 bytes

# Python handles this without any special type
print(type(d))                # <class 'int'>  - still just "int"

As you can see, sys.getsizeof() reveals that Python integers grow in memory as the value increases. A small number like 42 uses 28 bytes, while 2 ** 10000 (a 3011-digit number) uses over 1 KB.

Comparison with C++ Integer Types

In C++, integer types have fixed sizes and maximum values. If a computation exceeds the maximum, the result silently overflows and wraps around — a common source of bugs.

C++ Fixed-Size Types

// C++ integer limits
#include <climits>
#include <iostream>

int main() {
    // int: 4 bytes, max 2,147,483,647
    std::cout << sizeof(int) << std::endl;        // 4
    std::cout << INT_MAX << std::endl;             // 2147483647

    // long long: 8 bytes, max 9.2 x 10^18
    std::cout << sizeof(long long) << std::endl;   // 8
    std::cout << LLONG_MAX << std::endl;           // 9223372036854775807

    // OVERFLOW! Silent wraparound
    int x = INT_MAX;
    std::cout << x + 1 << std::endl;  // -2147483648 (BUG!)
}

Python — No Limit

import sys

x = 9_223_372_036_854_775_807  # Same as C++ LLONG_MAX
print(x + 1)              # 9223372036854775808 - still fine!

# Python can go far beyond any C++ type
x = 10 ** 100             # A googol - 101 digits
print(type(x))            # <class 'int'>

Arithmetic Operators

a, b = 17, 5

print(a + b)    # 22   Addition
print(a - b)    # 12   Subtraction
print(a * b)    # 85   Multiplication
print(a / b)    # 3.4  True division (always returns float)
print(a // b)   # 3    Floor division
print(a % b)    # 2    Modulo
print(a ** b)   # 1419857  Exponentiation
In C++ and Java, 17 / 5 performs integer division yielding 3. In Python, / always performs true division. Use // for floor division.

Precedence and Associativity of Arithmetic Operators

Operator precedence determines which operator is evaluated first when multiple operators appear in one expression. Associativity determines the direction of evaluation when two operators share the same precedence level. Python's arithmetic operators, ordered from highest to lowest precedence:

PrecedenceOperator(s)DescriptionAssociativity
1 (highest)**ExponentiationRight-to-left
2+x  -xUnary plus / unary minusRight-to-left
3*  /  //  %Multiplication, division, floor division, moduloLeft-to-right
4 (lowest)+  -Addition, subtractionLeft-to-right
# --- Precedence examples ---

# ** before * before +
print(2 + 3 * 4 ** 2)   # 50  →  2 + 3 * 16  →  2 + 48
print((2 + 3) * 4 ** 2) # 80  →  5 * 16

# / and // before +
print(10 + 6 / 2)        # 13.0  →  10 + 3.0
print(10 + 6 // 2)       # 13    →  10 + 3

# Unary minus applies before **... mostly (see below)
print(-2 ** 2)            # -4  (** first, then unary -: -(2**2))
print((-2) ** 2)          # 4   (parentheses force left side first)
Gotcha — unary minus and **: Even though unary minus is listed at a higher precedence row, Python parses -2 ** 2 as -(2 ** 2) = -4, not (-2) ** 2 = 4. This matches mathematical convention and is the same behavior as C++. Always use parentheses when negating a base in an exponentiation.
# --- Right-to-left associativity of ** ---
# a ** b ** c  is evaluated as  a ** (b ** c)
print(2 ** 3 ** 2)    # 512  →  2 ** (3**2)  =  2 ** 9  =  512
print((2 ** 3) ** 2)  # 64   →  (2**3) ** 2  =  8 ** 2  =  64

# --- Left-to-right associativity of +, -, *, / ---
# a - b - c  is evaluated as  (a - b) - c
print(20 - 5 - 3)     # 12   →  (20 - 5) - 3  =  15 - 3
print(20 - (5 - 3))   # 18   →  different result!

# True division is also left-to-right
print(100 / 5 / 2)    # 10.0  →  (100 / 5) / 2  =  20.0 / 2

C++ Arithmetic Precedence

// C++ precedence is very close to Python
// ** does not exist in C++ — use pow()
#include <cmath>

int a = 2 + 3 * 4;     // 14  (same as Python)
int b = 20 - 5 - 3;    // 12  (same left-to-right)

// No ** operator in C++
double x = pow(2, pow(3, 2));  // 512 (manual right-assoc)
double y = -pow(2, 2);         // -4  (same as Python -2**2)

Python Arithmetic Precedence

# Python adds ** with right-to-left associativity
a = 2 + 3 * 4       # 14
b = 20 - 5 - 3      # 12  (left-to-right)

# ** is a first-class operator, right-associative
x = 2 ** 3 ** 2     # 512  (right-to-left)
y = -2 ** 2         # -4   (** before unary -)

# Use parentheses to override any precedence
z = (2 ** 3) ** 2   # 64

Readable Numeric Literals — Underscore Separators

Python allows underscores (_) inside numeric literals as visual separators. They are ignored by the interpreter and exist only to make large numbers easier to read:

# Without underscores (hard to read)
population = 8100000000
budget = 4500000.75

# With underscores (much clearer)
population = 8_100_000_000
budget = 4_500_000.75

print(population)   # 8100000000  (underscore is not stored)
print(budget)       # 4500000.75

# Works with any numeric type
binary  = 0b1010_0001_1100_0011
hex_val = 0xFF_EC_DA
octal   = 0o17_55
big_int = 10_000_000_000_000

# Also works with floats and scientific notation
pi_approx = 3.14_15_92
avogadro  = 6.022_140_76e23
Underscores in numbers were introduced in Python 3.6 (PEP 515). C++ added a similar feature (1'000'000) in C++14, but uses single quotes instead.

Number Bases: Binary, Octal, Decimal, and Hexadecimal

Python supports four integer bases. Decimal (base 10) is the default numbering system — any integer literal without a prefix is decimal.

BasePrefixDigitsExampleDecimal Value
Binary (2)0b or 0B0–10b101010

# Writing integers in different bases
decimal = 255
binary  = 0b11111111     # 255 in binary
octal   = 0o377          # 255 in octal
hexa    = 0xFF           # 255 in hexadecimal

# They are all the same integer
print(decimal == binary == octal == hexa)   # True
print(type(binary))   # <class 'int'>  — always stored as int

Converting Between Bases

Use built-in functions to convert an integer to a string in a different base, or parse a string in a given base back to an integer:

n = 255

# int → string in different bases
print(bin(n))     # '0b11111111'
print(oct(n))     # '0o377'
print(hex(n))     # '0xff'

# Remove the prefix if needed
print(bin(n)[2:])   # '11111111'
print(hex(n)[2:])   # 'ff'

# String → int  (specify the base)
print(int('11111111', 2))    # 255   (binary string → int)
print(int('377', 8))         # 255   (octal string → int)
print(int('FF', 16))         # 255   (hex string → int)
print(int('255', 10))        # 255   (decimal string → int)

# int() auto-detects base when given a prefix and base=0
print(int('0b1010', 0))     # 10
print(int('0o17', 0))       # 15
print(int('0xFF', 0))       # 255

Default numbering system: Python always uses decimal (base 10) by default. When you print() any integer, it is displayed in decimal, regardless of how it was written. The prefixes 0b, 0o, and 0x are only for literal notation and output formatting.

2.3 Strings

Strings are immutable sequences of Unicode characters. Python does not have a separate char type; a single character is simply a string of length one.

name = "Ada Lovelace"
initial = name[0]           # "A"
length = len(name)          # 12

upper = name.upper()        # "ADA LOVELACE"
parts = name.split()        # ["Ada", "Lovelace"]
joined = "-".join(parts)    # "Ada-Lovelace"

Indexing and Slicing

Python supports powerful indexing using square brackets []. You can use positive indices (counting from the start at 0) or negative indices (counting backwards from the end at -1).

text = "PYTHON"
#        P  Y  T  H  O  N
#        0  1  2  3  4  5    ← positive index
#       -6 -5 -4 -3 -2 -1   ← negative index

# Positive indexing (from the start)
print(text[0])     # "P"  - first character
print(text[3])     # "H"  - fourth character
print(text[5])     # "N"  - last character

# Negative indexing (from the end)
print(text[-1])    # "N"  - last character
print(text[-2])    # "O"  - second to last
print(text[-6])    # "P"  - first character

Slicing extracts a substring using the syntax string[start:stop:step]. The start is inclusive, stop is exclusive, and step is optional.

text = "PYTHON"

# Basic slicing [start:stop]
print(text[0:3])    # "PYT"  - characters 0, 1, 2
print(text[2:5])    # "THO"  - characters 2, 3, 4

# Omit start or stop
print(text[:3])     # "PYT"  - from beginning to index 3
print(text[3:])     # "HON"  - from index 3 to end
print(text[:])      # "PYTHON" - full copy

# Negative indices in slicing
print(text[-3:])    # "HON"  - last 3 characters
print(text[:-2])    # "PYTH" - everything except last 2

# Step (stride)
print(text[::2])    # "PTO"  - every 2nd character
print(text[1::2])   # "YHN"  - every 2nd character starting at 1
print(text[::-1])   # "NOHTYP" - reversed string!
print(text[::-2])   # "NHY"  - reversed, every 2nd character

Slicing is safe — it never raises an error for out-of-range indices. Python silently clips them to the valid range:

text = "PYTHON"

print(text[0:100])   # "PYTHON" - stop clipped to len(text)
print(text[-100:3])  # "PYT"    - start clipped to 0

# But direct indexing DOES raise an error
# text[100]  → IndexError: string index out of range
Key insight: Indexing and slicing work identically on all Python sequences — strings, lists, and tuples. Learn it once, use it everywhere. In C++, std::string::substr(pos, len) takes a position and a length, while Python slicing takes start and stop positions.

2.4 Booleans and Comparison Operators

The bool type has exactly two instances: True and False. Many values have an implicit truth value: zero, empty collections, and None are falsy; everything else is truthy.

# Comparison operators (same as C++/Java)
print(5 > 3)       # True
print(5 == 5.0)    # True - cross-type comparison

# Boolean operators (words, not symbols)
print(True and False)   # False
print(True or False)    # True
print(not True)         # False

# Chained comparisons (unique to Python)
x = 7
print(1 < x < 10)  # True - equivalent to (1 < x) and (x < 10)
Python uses and, or, not instead of &&, ||, !.

Short-Circuit Evaluation

Python's and and or operators use short-circuit evaluation — they stop evaluating as soon as the result is determined. This is the same concept as && and || in C++, but with an important twist: Python returns the actual value that decided the result, not just True or False.

and — Short-Circuits on First Falsy

If the left operand is falsy, return it immediately (the right operand is never evaluated). Otherwise return the right operand.

or — Short-Circuits on First Truthy

If the left operand is truthy, return it immediately (the right operand is never evaluated). Otherwise return the right operand.

# ── Short-circuit with `and` ──
# If the first value is falsy, Python returns it immediately
# and NEVER looks at the second value

print(0 and 42)           # 0     (0 is falsy → returned immediately)
print("" and "hello")     # ""    ("" is falsy → returned immediately)
print(None and 100)       # None  (None is falsy → returned immediately)

# If the first value is truthy, Python must check the second value
print(1 and 42)           # 42    (1 is truthy → return second value)
print("hi" and "hello")   # "hello"

# ── Short-circuit with `or` ──
# If the first value is truthy, Python returns it immediately
# and NEVER looks at the second value

print(42 or 0)            # 42    (42 is truthy → returned immediately)
print("hello" or "")      # "hello"

# If the first value is falsy, Python must check the second value
print(0 or 42)            # 42    (0 is falsy → return second value)
print("" or "default")    # "default"
print(None or "fallback") # "fallback"

Proving Short-Circuit: Skipped Evaluation

We can prove that the second operand is never evaluated by using a function with a side effect:

def expensive():
    print("  expensive() was called!")
    return 42

# With `and`: if first is falsy, expensive() is NEVER called
print("Test 1:")
result = 0 and expensive()
print(f"  result = {result}")
# Output:
#   Test 1:
#   result = 0               ← expensive() was NOT called

print("Test 2:")
result = 1 and expensive()
print(f"  result = {result}")
# Output:
#   Test 2:
#   expensive() was called!  ← expensive() WAS called
#   result = 42

# With `or`: if first is truthy, expensive() is NEVER called
print("Test 3:")
result = "exists" or expensive()
print(f"  result = {result}")
# Output:
#   Test 3:
#   result = exists           ← expensive() was NOT called

Practical Uses

Short-circuit evaluation is not just a curiosity — Python programmers use it every day:

# 1. Default values (very common pattern)
username = input_name or "Anonymous"
# If input_name is empty/""/None → use "Anonymous"

# 2. Safe access - avoid division by zero
x = 10
y = 0
safe = y != 0 and x / y    # y is 0 → False, x/y never computed
print(safe)                  # False (no ZeroDivisionError!)

# 3. Conditional function call
data = None
result = data and data.process()  # data is None → process() never called
# This avoids: AttributeError: 'NoneType' has no attribute 'process'

# 4. First available value from multiple sources
config = None
env_var = ""
default = "localhost"
host = config or env_var or default
print(host)  # "localhost" (first truthy value wins)
C++ comparison: C++ && and || also short-circuit, but they always return true or false (a boolean). Python returns the actual operand that determined the result, which makes patterns like value or default possible.

2.5 Membership Operators

Python provides two membership operators that test whether a value exists inside a container. They work on any iterable — strings, lists, tuples, sets, and dictionaries — and always return a bool.

OperatorMeaningReturns
x in sTrue if x is found in sbool
x not in sTrue if x is not found in sbool

Strings — Substring Test

On strings, in checks for a substring, not just a single character.

text = "Hello, Python!"

print("Python" in text)       # True
print("python" in text)       # False  (case-sensitive)
print("lo" in text)           # True   (substring match)
print("z" in text)            # False
print("" in text)             # True   (empty string is always found)

# not in is the readable alternative to 'not (x in s)'
print("Java" not in text)     # True
print("Hello" not in text)    # False

Lists and Tuples — Element Test

On lists and tuples, in performs a linear search from left to right (O(n) time).

primes = [2, 3, 5, 7, 11, 13]

print(5 in primes)        # True
print(4 in primes)        # False
print(100 not in primes)  # True

# Works on tuples identically
coords = (10, 20, 30)
print(20 in coords)       # True

# Checks for the whole element, not sub-lists
nested = [[1, 2], [3, 4]]
print([1, 2] in nested)   # True   (finds the sublist as one element)
print(1 in nested)        # False  (1 is not a top-level element)

Dictionaries — Key Test

On dictionaries, in tests for keys, not values. Key lookup uses a hash table, so it runs in O(1) average time.

config = {"host": "localhost", "port": 8080, "debug": True}

print("host" in config)        # True   (key exists)
print("localhost" in config)   # False  (values are NOT checked)
print("timeout" not in config) # True   (key absent)

# To check values, iterate .values()
print(8080 in config.values())  # True

# To check key-value pairs, iterate .items()
print(("port", 8080) in config.items())  # True

Sets — O(1) Membership Test

Sets use a hash table internally, making in O(1) average time — the most efficient container for repeated membership tests.

allowed = {"admin", "editor", "viewer"}

print("admin" in allowed)      # True
print("guest" in allowed)      # False
print("guest" not in allowed)  # True

# Prefer sets over lists when you only need membership testing
# List:  O(n) — scans every element
# Set:   O(1) — hash lookup

C++ Equivalent

// C++ has no built-in 'in' operator.
// Use algorithms or container methods.
#include <algorithm>
#include <set>
#include <unordered_set>
#include <string>

// Vector / list  — linear search O(n)
std::vector<int> v = {2, 3, 5, 7};

// set / unordered_set  — O(log n) / O(1)
std::unordered_set<std::string> roles = {"admin", "editor"};

// String substring
std::string s = "Hello, Python!";
bool sub = s.find("Python") != std::string::npos;

Python Equivalent

# Python — uniform 'in' operator across all containers

# List  (linear scan)
v = [2, 3, 5, 7]
found = 5 in v

# Set  (O(1) hash lookup)
roles = {"admin", "editor"}
ok = "admin" in roles

# String substring
s = "Hello, Python!"
sub = "Python" in s

# All three look identical — Python picks
# the right algorithm based on the type
Key difference from C++: Python's in is a true operator with consistent syntax across all container types. C++ requires different algorithms or methods for each container (std::find, .count(), .find()), and there is no single unified syntax.

2.6 Identity Operators

Identity operators test whether two names refer to the exact same object in memory, not just whether their values are equal. This is the difference between asking "are these the same thing?" versus "do these have the same value?"

OperatorMeaningEquivalent to
x is yTrue if x and y point to the same objectid(x) == id(y)

is vs == — Identity vs Equality

== calls the object's __eq__ method to compare values. is bypasses that entirely and compares memory addresses using id().

a = [1, 2, 3]
b = [1, 2, 3]   # same value, different object
c = a           # c is an alias — same object as a

print(a == b)   # True   (values are equal)
print(a is b)   # False  (different objects in memory)

print(a == c)   # True   (values are equal)
print(a is c)   # True   (same object — c is just another name for a)

# Mutating through one alias affects the other
c.append(4)
print(a)        # [1, 2, 3, 4]  — a changed too!
print(id(a) == id(c))  # True
Rule of thumb: Use == to compare values. Reserve is for three specific cases: checking against None, True, and False as singletons.

The Correct Way to Test for None

None is a singleton — there is exactly one None object in all of Python. The idiomatic and recommended way to test for it is with is, not ==. PEP 8 (Python's style guide) explicitly requires this.

result = None

# Correct (PEP 8 style)
if result is None:
    print("no result yet")

if result is not None:
    print("got a result:", result)

# Technically works but wrong style
if result == None:     # avoid — relies on __eq__
    pass
if not result:         # WRONG — also catches 0, "", [], False!
    pass

Integer Caching — A CPython Implementation Detail

CPython (the standard Python interpreter) caches all integer objects in the range -5 to 256 and interns short strings. This means two variables assigned the same small integer will is-compare as True — but this is an implementation detail, not a language guarantee.

# Small integers are cached — same object reused
a = 100
b = 100
print(a is b)    # True  (both point to the cached int object 100)
print(id(a), id(b))  # same id

# Large integers are NOT cached — new objects each time
x = 1000
y = 1000
print(x is y)    # False  (two separate int objects with value 1000)
print(x == y)    # True   (values are equal)

# Same applies to strings in some cases
s1 = "hello"
s2 = "hello"
print(s1 is s2)  # True in CPython (interned short strings)

long1 = "hello world!"
long2 = "hello world!"
print(long1 is long2)  # May be False (not guaranteed)
Never rely on is for integer or string equality in real code. The caching behaviour is a CPython optimisation that can change between versions and implementations (PyPy, Jython, etc.). Always use == to compare values.

Comparison with C++ Pointer Comparison

C++ Pointer / Reference Identity

// In C++, identity comparison means comparing
// pointer addresses
#include <iostream>

int main() {
    int a = 42, b = 42;
    int* p = &a;
    int* q = &a;  // same object
    int* r = &b;  // different object, same value

    std::cout << (p == q);  // 1 (true)  — same address
    std::cout << (p == r);  // 0 (false) — different address
    std::cout << (*p == *r); // 1 (true)  — same value

    // nullptr check
    int* ptr = nullptr;
    if (ptr == nullptr) { ... }  // C++ idiom
}

Python Identity Operators

# Python 'is' is the high-level equivalent of
# comparing pointer addresses in C++

a = [1, 2]
b = [1, 2]  # same value, different object
c = a       # alias — same object

print(a is c)    # True  (same "address")
print(a is b)    # False (different "address")
print(a == b)    # True  (same value)

# None check — Python idiom
result = None
if result is None:   # equivalent to: ptr == nullptr
    pass

# id() gives the memory address
print(id(a))    # e.g. 140234567890
print(id(c))    # same number as id(a)

2.7 Type Conversion

Python distinguishes between two kinds of type conversion: implicit (done automatically by the interpreter) and explicit (done intentionally by the programmer using built-in functions). Understanding the boundary between the two prevents subtle bugs.

Implicit Type Conversion (Coercion)

Implicit conversion happens automatically when Python evaluates an expression involving two different numeric types. Python always promotes the value to the wider type so no information is lost. The promotion hierarchy is:

bool → int → float → complex

# int + float → float  (int is promoted to float)
result = 10 + 2.5
print(result)           # 12.5
print(type(result))     # <class 'float'>

# int + complex → complex  (int is promoted to complex)
result = 3 + (1 + 2j)
print(result)           # (4+2j)
print(type(result))     # <class 'complex'>

# bool is a subclass of int: True == 1, False == 0
print(True + 5)         # 6   (True is treated as 1)
print(False + 10)       # 10  (False is treated as 0)
print(True * 3.14)      # 3.14
print(sum([True, True, False, True]))  # 3  (counting Trues)
Python is not JavaScript: Python will not implicitly convert between unrelated types such as str and int. "3" + 5 raises a TypeError in Python, while JavaScript silently produces "35". Python's implicit coercion is limited strictly to the numeric promotion hierarchy.
# These raise TypeError — Python refuses to guess
# "3" + 5      → TypeError: can only concatenate str (not "int") to str
# 10 + "20"    → TypeError: unsupported operand type(s) for +: 'int' and 'str'
# True + "yes" → TypeError: unsupported operand type(s) for +: 'int' and 'str'

C++ Implicit Conversions

// C++ has many implicit conversions
// that can silently lose data
int   i = 3.99;    // double → int: truncated to 3!
float f = i;       // int → float: OK
bool  b = 42;      // int → bool: true (non-zero)
char  c = 300;     // int → char: silently overflows!

// Numeric to bool is implicit
if (42) { }        // OK in C++
int n = true;      // bool → int: 1

Python Implicit Conversions

# Python only promotes within the numeric hierarchy

x = 3 + 0.99       # int + float  → float:   3.99
z = 1 + (0+1j)     # int + complex → complex: (1+1j)
n = True + 6       # bool + int   → int:     7

# Python does NOT do these silently:
# int  → str  (needs str(x))
# float → int  (needs int(x))
# str  → bool  based on content

Explicit Type Conversion (Type Casting)

Explicit conversion uses Python's built-in constructor functions. Unlike C++ casts, these are ordinary callable objects that construct a new object of the target type.

int() — Converting to Integer

# float → int: truncates toward zero (does NOT round)
print(int(3.9))      # 3   (not 4!)
print(int(-3.9))     # -3  (not -4!)
print(int(0.999))    # 0

# str → int: the string must be a plain integer literal
print(int("42"))     # 42
print(int("-100"))   # -100

# Optional base argument for non-decimal strings
print(int("FF", 16))      # 255  (hexadecimal)
print(int("1010", 2))     # 10   (binary)
print(int("17", 8))       # 15   (octal)

# bool → int
print(int(True))     # 1
print(int(False))    # 0

# These raise ValueError:
# int("3.14")   → ValueError: invalid literal for int() with base 10
# int("12 kg")  → ValueError: invalid literal for int() with base 10
# int("")        → ValueError: invalid literal for int() with base 10

float() — Converting to Float

print(float(7))         # 7.0
print(float("3.14"))    # 3.14
print(float("1e5"))     # 100000.0  (scientific notation is OK)
print(float("inf"))     # inf
print(float("nan"))     # nan
print(float(True))      # 1.0

# Raises ValueError:
# float("3,14")   → ValueError (comma is not a decimal separator)
# float("12 kg")  → ValueError

str() — Converting to String

print(str(42))           # "42"
print(str(3.14))         # "3.14"
print(str(True))         # "True"
print(str(None))         # "None"
print(str([1, 2, 3]))    # "[1, 2, 3]"

# str() never raises — every Python object has a string representation
# because every class inherits __str__() from object

bool() — Truthiness Rules

Every Python object has a truth value. The following are falsy; everything else is truthy:

# Falsy
print(bool(0))        # False
print(bool(0.0))      # False
print(bool(""))       # False
print(bool(None))     # False
print(bool([]))       # False

# Truthy — note the surprises
print(bool(1))        # True
print(bool(-1))       # True  (any non-zero number)
print(bool("0"))      # True  ("0" is a non-empty string!)
print(bool(" "))      # True  (whitespace is also truthy)
print(bool([0]))      # True  (list with one element, even 0)
Common pitfall: bool("False") is True — any non-empty string is truthy regardless of its content. To check if a string represents a false boolean, compare explicitly: val.lower() == "false".

Comparison with C++ Casts

C++ Cast Syntax

// Preferred: named cast
int i = static_cast<int>(3.9);       // 3 (truncates)
double d = static_cast<double>(5);   // 5.0

// Old C-style cast (same effect)
int n = (int)3.9;   // 3

// reinterpret_cast — raw bit reinterpretation
// (no direct Python equivalent)
float f = 3.14f;
int bits = *reinterpret_cast<int*>(&f);

Python Explicit Conversion

# Python — just function calls, no special syntax
i = int(3.9)      # 3   (truncates toward zero)
d = float(5)      # 5.0
s = str(42)       # "42"

# Python has no reinterpret_cast equivalent.
# Use struct for raw byte interpretation:
import struct
bits = struct.unpack('I', struct.pack('f', 3.14))[0]
