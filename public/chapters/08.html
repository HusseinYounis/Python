8.1 try / except / else / finally

Python's exception handling is similar to Java's try/catch/finally. C++ uses try/catch without an else clause.

try:
    value = int(input("Enter a number: "))
    result = 100 / value
except ValueError:
    print("Not a valid integer")
except ZeroDivisionError:
    print("Cannot divide by zero")
else:
    # Runs only if NO exception occurred
    print(f"Result: {result}")
finally:
    # Always runs (cleanup)
    print("Done")
The else clause is unique to Python. It is preferred over putting code in the try block when that code should only run if no exception was raised.

8.2 Exception Hierarchy

All exceptions inherit from BaseException. Most user-catchable exceptions inherit from Exception.

BaseException
 +-- SystemExit
 +-- KeyboardInterrupt
 +-- Exception
      +-- ValueError
      +-- TypeError
      +-- KeyError
      +-- IndexError
      +-- FileNotFoundError
      +-- IOError
      +-- AttributeError
      +-- RuntimeError
      +-- StopIteration
Never catch bare except: or except BaseException in production code — it will silently swallow KeyboardInterrupt and SystemExit.

8.3 Custom Exceptions

class InsufficientFundsError(Exception):
    """Raised when a withdrawal exceeds the balance."""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(
            f"Cannot withdraw ${amount:.2f}; "
            f"balance is ${balance:.2f}"
        )

class BankAccount:
    def __init__(self, balance=0):
        self._balance = balance

    def withdraw(self, amount):
        if amount > self._balance:
            raise InsufficientFundsError(self._balance, amount)
        self._balance -= amount
        return self._balance

# Usage
try:
    acct = BankAccount(100)
    acct.withdraw(150)
except InsufficientFundsError as e:
    print(e)  # Cannot withdraw $150.00; balance is $100.00

8.4 Context Managers (with Statement)

The with statement ensures resources are properly released, similar to Java's try-with-resources and C++'s RAII pattern.

# File handling - guaranteed close
with open("data.txt", "r") as f:
    content = f.read()
# f is automatically closed here

# Creating custom context managers
from contextlib import contextmanager

@contextmanager
def timer(label):
    import time
    start = time.perf_counter()
    yield
    elapsed = time.perf_counter() - start
    print(f"{label}: {elapsed:.4f}s")

with timer("Sort"):
    sorted(range(1_000_000, 0, -1))

Class-Based Context Manager

class DatabaseConnection:
    def __enter__(self):
        self.conn = connect_to_db()
        return self.conn

    def __exit__(self, exc_type, exc_val, exc_tb):
        self.conn.close()
        return False  # Don't suppress exceptions

with DatabaseConnection() as conn:
    conn.execute("SELECT * FROM users")

8.5 EAFP vs LBYL

LBYL (C++/Java style)

if key in dictionary:
    value = dictionary[key]
else:
    value = default

EAFP (Pythonic)

try:
    value = dictionary[key]
except KeyError:
    value = default

Python idiom prefers EAFP. Even simpler: value = dictionary.get(key, default)
